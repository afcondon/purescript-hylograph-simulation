PSD3 Visualization Test

TickCoordinator → Consumers → PSD3 AST → D3 DOM

Architecture: This demo shows the full unified stack:

Coordinator (single RAF loop) → simulationConsumer (updates positions) → PSD3 AST (declarative spec) → D3 Interpreter (DOM updates via data join)

The visualization is defined declaratively using A.joinData and attributes like Attr.attr "cx" _.x showNumD. The interpreter handles the enter/update/exit pattern automatically.
